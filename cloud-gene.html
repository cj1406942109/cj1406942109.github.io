loginMain.js(登录页面js主模块)

详细信息：

  1. 由require.js作为主模块引入到login.html文件

  2. 配置和引入login.html文件中需要用到的第三方依赖，可以看到具体模块对应的路径

  3. 手动引导angularjs应用loginApp 

        function(j,angular){
            //angularjs获取document元素 
            angular.element(document).ready(function(){
                angular.bootstrap(document,['loginApp']);
            });
        }

loginApp.js(登录页面angularjs模块和指令)

详细信息：

  1. 定义loginApp模块

  2. 需要依赖ui.router和superApp.superService模块

  3. 对loginApp模块进行相关配置

    • 使用$logProvider.debugEnabled(true);开启调试级消息

  4. 定义superValidate指令

    • 依赖$log服务

    • 设置指令在dom中的声明方式为ACE

    • 设置链接函数，用于更新dom和注册事件监听。

      • 将指令注册元素用myvalidate-wrap类包裹，并在元素后添加myvalidate-wrap-tips类用于显示提示信息。

      • 添加hover事件监听，鼠标移入时，如果元素已被点击，并且输入信息不合法，则显示提示信息，鼠标移出时，隐藏提示信息。

loginController.js(登录页面控制器)

详细信息：

  1. 定义loginApp模块的控制器loginController

  2. 控制器需要依赖服务$log、ajaxService和tooService，其中ajaxService和toolService为自定义模块，具体内容见loginMain.js文件的superApp.superService。

  3. 在作用域对象中添加formEntity对象属性 

    $scope.formEntity = {

  4. 控制器需要依赖服务$rootscope、$log、$state、pageFactory、ajaxService和tooService，其中ajaxService和toolService为自定义模块，具体内容见loginMain.js文件的superApp.superService，pageFactory为自定义工厂函数，具体内容见home.Main.js文件的pageFactory。

    • 根据项目名和项目ID判断是添加逻辑还是修改逻辑

      • 修改逻辑，调用方法GetXMEnitty()

      • 添加逻辑，设置formEntity为空

    • $state.go("userconsole/userProHome");

    • 如果是添加逻辑，调用Insert_Project()方法
